## Design Principles

### Target Experience

- **Professional** - Clean, modern, GitHub/Linear/Notion aesthetic
- **Efficient** - Common actions accessible within 1-2 clicks
- **Focused** - Minimal chrome, maximize content area
- **Discoverable** - Clear navigation, helpful empty states
- **Consistent** - Unified patterns across all views

### Key User Flows

1. **First-time user:** Import CSV â†’ Map columns â†’ Preview â†’ View timeline
2. **Regular update:** Quick re-import with saved profile â†’ View updated timeline
3. **Scenario planning:** Create branch â†’ Modify data â†’ Compare branches
4. **Audit trail:** Search item â†’ View version history â†’ Revert change

### Content Density

- **Primary focus:** Timeline canvas (should occupy 70%+ of viewport)
- **Secondary:** Controls, filters, metadata panels
- **Tertiary:** Navigation, branding, status indicators

- **Simple mental model** - One view at a time, clear separation
- **Maximum vertical space** - Tabs take minimal height
- **Familiar pattern** - Users understand tabs immediately
- **Easy to implement** - Standard React pattern

### Cons
- **No persistent context** - Branch selector/status changes per tab
- **Context switching** - Can't see timeline while comparing branches
- **Limited multi-tasking** - Can't reference timeline while importing

### Best Use Cases
- Users follow linear workflows (import â†’ view â†’ compare)
- Single monitor setups
- Mobile/tablet use

## Recommendations

### Recommended Approach: **Option C (GitHub Style) + Enhancements**

**Why this works best:**

1. **Familiar to target users** - GitHub-style navigation is widely understood
2. **Clean & modern** - Matches 2024/2025 design trends
3. **Flexible** - Command palette enables power users, visual UI for beginners
4. **Great responsive story** - Collapses gracefully to mobile
5. **Room to grow** - Easy to add features without cluttering UI

### Key Design Decisions

#### Navigation
- **Top horizontal tabs** for main views (Import, Timeline, Branches, History)
- **Command bar** below header for contextual actions
- **Branch selector** always visible in top-right
- **Export** as dropdown menu (PNG, PDF, CSV, etc.)

#### Timeline View Priorities
1. **Maximize canvas space** - Controls should be collapsible
2. **Persistent legend** - Always show color coding
3. **Floating zoom controls** - Bottom-right corner (like Google Maps)
4. **Selection panel** - Bottom drawer or right sidebar (toggleable)

#### Color Palette
- **Tasks:** Blue (#2196F3)
- **Milestones:** Green (#4CAF50)
- **Releases:** Orange (#FF9800)
- **Meetings:** Purple (#9C27B0)
- **UI Chrome:** Neutral grays (#F5F5F5, #E0E0E0, #333)

#### Typography
- **Headings:** 16-24px, semibold (Inter or SF Pro)
- **Body:** 14px, regular
- **Monospace:** 13px (for IDs, dates)

#### Spacing
- **Tight:** 4-8px (within components)
- **Medium:** 12-16px (between sections)
- **Loose:** 20-24px (between major areas)

### Implementation Priority

**Phase 1:** Basic layout with top tabs
**Phase 2:** Timeline view with controls
**Phase 3:** Command bar and keyboard shortcuts
**Phase 4:** Polish, animations, empty states
